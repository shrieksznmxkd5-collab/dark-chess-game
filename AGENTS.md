# 三、明棋与暗棋

以下6枚棋子从开局开始正面显示，不参与随机：

* 红帅；
* 黑将；
* 红方两枚士；

它们固定放在标准中国象棋开局位置。

除将、帅和四枚士之外，其余26枚棋子全部参与随机洗牌。

包括：

红方：

* 车×2；
* 马×2；

* 炮×2；
* 兵×5。

黑方：

* 车×2；
* 马×2；
* 象×2；
* 炮×2；
* 卒×5。

这26枚棋子：

1. 随机打乱；
2. 全部背面朝上；
3. 随机放在除将、帅、士之外的26个标准象棋开局位置；
4. 棋盘中原本为空的位置继续为空。

不能把暗棋随机放到全部90个交叉点，只能在26个标准开局棋位之间随机交换。

# 四、暗棋同时拥有两套身份

每一枚暗棋必须同时保存以下信息：

```js
{
  id: "piece-01",

  // 被隐藏的真实信息
  realType: "cannon",
  realCamp: "red",

  // 翻面前使用的临时信息
  initialRole: "rook",
  initialCamp: "black",

  // 状态信息
  faceUp: false,
  alive: true,
  hasActed: false,

  // 位置
  row: 0,
  col: 0,

  // 象翻面时所在的棋盘半区
  lockedRiverSide: null
}
```

字段含义：

* `realType`：真实棋子种类；
* `realCamp`：真实红黑阵营；
* `initialRole`：该开局位置原本对应的棋子职能；
* `initialCamp`：该开局位置属于红方还是黑方；
* `faceUp`：是否已经翻面；
* `hasActed`：是否已经完成首次行动；
* `lockedRiverSide`：象翻面后被限制在哪一侧活动。

暗棋在翻面前：

* 不使用真实身份；
* 不使用真实阵营；
* 完全按照开局位置赋予的临时身份行动。

有效身份获取逻辑应类似：

```js
function getActiveType(piece) {
  return piece.faceUp ? piece.realType : piece.initialRole;
}

function getActiveCamp(piece) {
  return piece.faceUp ? piece.realCamp : piece.initialCamp;
}
```

# 五、暗棋翻面前的职能

暗棋翻面之前，完全就是其开局位置原本对应的标准棋子。

例如：

* 位于车位的暗棋，第一次按照车行动；
* 位于马位的暗棋，第一次按照马行动；
* 位于象位的暗棋，第一次按照象行动；
* 位于炮位的暗棋，第一次按照炮行动；
* 位于兵卒位的暗棋，第一次按照兵卒行动。

翻面前的暗棋可以：

* 普通移动；
* 吃掉敌方棋子。

移动与吃子都必须完整遵守该临时棋子的标准规则，例如：

* 马受蹩马腿限制；
* 象受塞象眼限制；
* 炮吃子时必须隔一个炮架；
* 兵卒按照临时阵营决定前进方向；
* 未过河的兵卒不能横走；
* 车不能越过阻挡棋子。

暗棋翻面前的控制权由 `initialCamp` 决定：

* 红方标准开局区域的暗棋暂时由红方控制；
* 黑方标准开局区域的暗棋暂时由黑方控制。

---

# 六、首次行动和翻面

暗棋第一次完成合法移动或合法吃子后，必须立即翻面。

完整流程：

1. 玩家选择暗棋；
2. 按照 `initialRole` 判断合法落点；
3. 棋子执行移动或吃子；
4. 棋子到达目标位置；
5. 播放翻面动画；
6. 显示 `realType` 与 `realCamp`；
7. 设置 `faceUp = true`；
8. 设置 `hasActed = true`；
9. 从此失去 `initialRole`；
10. 后续完全按照真实身份行动；
11. 控制权转交给 `realCamp` 对应的玩家。

例如：

红方控制一枚临时职能为“车”的暗棋。

该暗棋第一次按照车的规则完成移动，翻面后显示为黑马。

那么：

* 刚才的车式移动有效；
* 棋子不回到原位置；
* 从翻面结束开始，它归黑方控制；
* 后续按照马的规则行动。

翻面后阵营发生变化，不撤销已经完成的移动或吃子。

翻面不会额外增加回合，完成行动后正常切换玩家。

---

# 七、未翻面的暗棋被吃

未翻面的暗棋可以被吃。

规则：

1. 按照暗棋当前的 `initialCamp` 判断敌我；
2. 被合法吃掉后直接从棋盘移除；
3. 不再翻面；
4. 不显示它的真实身份；
5. 不因为它隐藏的真实阵营而撤销吃子；
6. 将、帅、士和其他棋子都可以在合法情况下吃暗棋。

---

# 八、标准棋子移动规则

请实现标准中国象棋移动规则。

## 1. 车

* 横向或纵向任意距离；
* 路径中不能有其他棋子；
* 可以吃目标位置的敌方棋子。

## 2. 马

* 走“日”字；
* 受蹩马腿限制。

## 3. 炮

普通移动：

* 与车一样横向或纵向移动；
* 路径不能有棋子。

吃子：

* 攻击者与目标之间必须恰好隔一枚棋子作为炮架；
* 不能隔零枚或两枚以上。

## 4. 兵和卒

红兵：

* 向棋盘上方前进；
* 过河前只能向前一步；
* 过河后可以向前或左右一步；
* 不能后退。

黑卒：

* 向棋盘下方前进；
* 过河前只能向前一步；
* 过河后可以向前或左右一步；
* 不能后退。

暗棋翻面前，兵卒方向按照 `initialCamp` 判断。

翻面后，按照 `realCamp` 判断。

## 5. 士

士从开局开始就是明棋。

规则：

* 只能斜走一格；
* 红士只能在红方九宫内；
* 黑士只能在黑方九宫内；
* 不能离开九宫；
* 可以合法吃明棋或暗棋。

## 6. 将和帅

规则：

* 只能在己方九宫内横向或纵向移动一格；
* 不能离开九宫；
* 可以合法吃明棋或暗棋；
* 保留将帅照面规则。

## 7. 象和相

暗面阶段：

* 按照开局象位对应的临时阵营活动；
* 走“田”字；
* 受塞象眼限制；
* 不能跨越楚河汉界。

翻面后：

1. 按照真实象/相身份走“田”字；
2. 继续受塞象眼限制；
3. 翻面时记录它当前位于哪一侧河岸；
4. 此后只能在翻面时所在的这一半棋盘内活动；
5. 永远不能跨越楚河汉界。

例如：

一枚红相在黑方半场翻面。

它仍属于红方并由红方控制，但是：

* 可以继续在黑方半场按象的方式移动；
* 不能跨河回到红方半场。

---

# 九、将帅安全

正常情况下，任何玩家完成行动后，自己的将或帅都不能处于敌方攻击范围内。

行动合法性需要检查：

1. 移动规则是否合法；
2. 是否吃了己方棋子；
3. 行动后自己的将帅是否安全；
4. 是否造成将帅照面。

如果普通行动导致己方将帅被攻击，则该步非法。

但是下面的“暗棋翻面延迟威胁”属于特殊例外。

---

# 十、将帅照面

保留标准中国象棋将帅照面规则。

红帅与黑将：

* 不能位于同一纵线；
* 且两者中间没有任何棋子阻挡。

任何移动、吃子或翻面后的局面，如果导致将帅直接照面，则该步非法。

将帅照面不适用延迟保护机制，必须立即阻止。

---

# 十一、暗棋翻出敌方并攻击己方将帅

这是本游戏最重要的特殊规则。

例如：

1. 当前是红方回合；
2. 红方操作一枚暗棋完成第一次移动；
3. 该暗棋翻面后显示为黑棋；
4. 该黑棋按真实身份计算，刚好可以攻击红帅。

此时：

* 红方刚才的行动依然有效；
* 棋子不回滚；
* 被吃棋子不恢复；
* 暗棋保持翻面；
* 该攻击暂时不判定为正式将军；
* 不允许黑方立即利用该攻击吃帅；
* 红方会在黑方完成下一步后获得一次调整机会。

该机制称为：

```text
延迟威胁 / 延迟将军保护
```

---

# 十二、延迟威胁触发条件

只有同时满足以下条件时才触发：

1. 行动开始前，当前玩家自己的将帅处于安全状态；
2. 当前玩家操作的是一枚未翻面的暗棋；
3. 暗棋第一次移动或吃子合法；
4. 暗棋翻面后属于对方阵营；
5. 正是这枚新翻面的棋子，按照真实身份新产生了对当前玩家将帅的攻击；
6. 没有造成将帅照面；
7. 没有其他普通敌方棋子同时产生新的非法攻击。

延迟威胁只冻结“新翻面棋子产生的这一次攻击”。

它不能用于逃避原本已经存在的将军。

---

# 十三、延迟威胁状态机

建议增加以下状态：

```js
const delayedThreat = {
  active: true,

  // 被保护、稍后获得调整机会的阵营
  protectedCamp: "red",

  // 当前即将行动、但不能正式将军的阵营
  restrictedCamp: "black",

  // 触发特殊规则的翻面棋子
  triggerPieceId: "piece-12",

  // restricted-turn 或 adjustment-turn
  phase: "restricted-turn"
};
```

游戏至少包含以下状态：

```text
normal
selected
animating
revealing
restricted-turn
adjustment-turn
game-over
```

---

# 十四、限制方回合

延迟威胁触发后，正常切换到新翻面棋子所属阵营。

例如：

红方翻出黑棋并形成潜在攻击，接下来进入黑方回合。

黑方可以：

* 移动刚翻出的黑棋；
* 移动其他黑棋；
* 吃红方普通棋子；
* 进行正常布局；
* 翻开其他暗棋。

但是黑方本回合不能：

1. 吃掉红帅；
2. 用其他棋子新形成正式将军；
3. 移开遮挡形成闪将；
4. 用炮、车、马或其他棋子主动产生新的将军；
5. 将触发棋子移动到另一个能够攻击红帅的位置；
6. 利用被冻结的潜在攻击直接吃帅。

重要说明：

* 触发延迟机制的那枚棋子如果原地保持不动，它原本对红帅的攻击暂时被冻结；
* 该攻击在黑方限制回合中不算正式将军；
* 黑方不能直接利用这条攻击吃掉红帅；
* 黑方如果移动触发棋子，则其新位置不能攻击红帅。

如果黑方没有任何符合限制条件的合法行动，则自动跳过黑方限制回合，直接进入红方调整回合。

---

# 十五、受保护方调整回合

黑方限制回合结束后，红方获得一个调整回合。

红方可以：

* 移动帅；
* 吃掉潜在威胁棋子；
* 用其他棋子遮挡攻击路线；
* 移动其他棋子解除攻击；
* 通过任何合法方式消除潜在威胁。

红方调整回合结束时，必须按照完整正常规则检查红帅安全。

此时被冻结的攻击恢复参与判断。

如果红方行动结束后，红帅仍处于任何黑棋攻击范围内，则该步非法，需要重新选择。

红方成功解除威胁后：

1. 清除 `delayedThreat`；
2. 恢复普通回合状态；
3. 之后黑方可以正常将军；
4. 游戏继续。

如果红方完全没有任何能够解除潜在威胁的合法着法：

1. 红方调整回合自动跳过；
2. 延迟保护结束；
3. 下一回合黑方恢复正常攻击权限；
4. 黑方仍需通过一次合法行动实际吃掉红帅才能获胜。

该规则对红黑双方完全对称。

---

# 十六、胜利条件

胜利只以将或帅被合法吃掉为准。

不使用以下胜负条件：

* 不以无棋可走直接判负；
* 不以困毙直接判负；
* 不以其他棋子全部被吃判负；
* 不以形成将军或将死自动结束。

只有将或帅从棋盘上被合法吃掉，游戏才结束。

如果普通回合一方完全没有合法着法，但将帅仍在：

* 自动跳过该方回合；
* 不直接判负。

---

# 十七、行动事务与状态回滚

所有棋局逻辑必须与动画表现分离。

每次行动使用事务式流程：

1. 保存行动前状态快照；
2. 校验基础移动规则；
3. 临时执行移动；
4. 执行吃子；
5. 如果是暗棋首次行动，执行翻面；
6. 重新计算控制权；
7. 检查将帅照面；
8. 检查普通将帅安全；
9. 检查是否触发延迟威胁；
10. 合法则提交状态；
11. 非法则从快照完整恢复。

快照至少保存：

* 所有棋子位置；
* 所有棋子存活状态；
* 翻面状态；
* 当前玩家；
* 被吃棋子；
* 延迟威胁状态；
* 游戏是否结束。

动画层不能直接更改棋局数据，只能根据逻辑层提交后的结果播放。

# 二十、棋子移动质感

棋子不能瞬间跳到目标位置。

普通移动动画流程：

```text
选中棋子
→ 棋子轻微抬起
→ 沿合法路径平滑移动
→ 移动路径留下残影
→ 棋子落下
→ 轻微回弹
→ 必要时翻面
→ 更新回合
```

参数建议：

* 普通移动时间：250至450毫秒；
* 棋子选中后放大至约1.04至1.07；
* 落地有轻微压缩和回弹；
* 不要使用夸张弹簧效果；
* 动画期间锁定其他棋盘输入。

移动路径：

* 车、炮、兵卒、士、将帅：按照实际路线移动；
* 马：使用轻微弧线或分段曲线表现“日”字；
* 象：使用柔和斜向弧线；
* 动画不能影响实际规则判定。

# 二十一、移动路径残影

棋子移动过程中，要在经过的路径留下逐渐消失的痕迹。

残影风格：

* 淡墨色运动轨迹；
* 少量金色微光；
* 2至5个逐渐透明的棋子轮廓；
* 不能做成激光或霓虹效果。

残影持续时间：

* 约600至1000毫秒；
* 越靠近起点越透明；
* 最终自然消散。

不同棋子：

* 车：直线墨迹；
* 炮：直线墨迹；
* 马：日字弧线或折线残影；
* 象：田字斜向轨迹；
* 炮吃子时，炮架位置短暂高亮。

动画结束后必须移除临时残影节点，避免内存累积。

建议使用：

* CSS transform；
* CSS opacity；
* Canvas覆盖层或临时DOM元素。

避免频繁修改 `top` 和 `left` 导致页面重排。

---

# 二十二、暗棋翻面动画

暗棋完成首次行动后：

1. 在目标位置短暂悬停；
2. 沿Y轴进行3D翻转；
3. 翻转接近90度时才切换正面；
4. 不能提前泄露真实身份；
5. 翻面完成后轻微落下。

反馈：

* 翻出当前行动方棋子：柔和暖金光；
* 翻出对方棋子：克制的冷色或暗红边缘光；
* 翻出强子时可以有略明显的低频震动；
* 不同棋子不要使用过度夸张的爆炸特效。

翻面后显示短暂提示，例如：

```text
揭示：黑方·马
控制权转移
```

---

# 二十三、吃子反馈

吃子动画顺序：

1. 被吃棋子轻微震动；
2. 被吃棋子缩小并淡出；
3. 攻击棋子移动到目标位置；
4. 目标点产生轻微墨点、木屑或冲击波；
5. 被吃棋子进入棋盘侧边记录区域。

炮吃子时：

* 炮架位置短暂发亮；
* 路径产生断续墨迹；
* 目标位置反馈略强于普通吃子。

---

# 二十四、合法落点反馈

选中棋子后：

* 棋子出现克制的高亮外圈；
* 合法空位显示柔和圆点；
* 可以吃的敌方棋子显示更明显的外框；
* 非法落点不显示；
* 不允许点击非法落点直接执行。

非法点击时：

* 棋子轻微回弹；
* 使用很轻的提示音；
* 显示简短原因，例如：

  * “马腿被阻挡”；
  * “炮吃子需要一个炮架”；
  * “士不能离开九宫”；
  * “该行动会导致将帅照面”；
  * “延迟保护期间不能将军”。

---

# 二十五、延迟威胁的界面表现

暗棋翻出敌方棋子并产生潜在攻击时：

不要显示正式“将军”。

显示：

```text
潜在威胁
该攻击暂时冻结
你将在下一回合获得一次调整机会
```

视觉效果：

* 触发棋子出现克制的暗红与金色波纹；
* 触发棋子到将帅之间的攻击路线短暂显示；
* 被威胁将帅出现琥珀色低频呼吸边框；
* 使用低沉预警音效；
* 不使用正式将军音效。

限制方回合显示：

```text
黑方行动
本回合不能对红帅形成新的将军
```

调整方回合显示：

```text
红方调整回合
请在本回合结束前解除潜在威胁
```

威胁解除时：

* 攻击路线化为淡墨消散；
* 将帅呼吸边框消失；
* 显示“威胁解除”；
* 恢复普通界面。

# 二十六、将军与胜利反馈

正常将军时：

* 将或帅出现低频红色呼吸光；
* 显示“将军”；
* 使用短促、克制的提示音；
* 当前玩家只能选择能够解除将军的合法行动。

将或帅被吃时：

1. 立即锁定棋盘操作；
2. 播放克制的胜负动画；
3. 显示胜利方；
4. 提供“重新开局”按钮；
5. 不自动刷新页面。

* 选中棋子；
* 普通移动；
* 棋子落地；
* 暗棋翻面；
* 吃子；
* 炮吃子；

# 二十八、性能要求

1. 优先使用CSS transform和opacity；
2. 动画时不要频繁触发布局重排；
3. 限制同屏粒子和残影数量；

5. 快速点击时锁定输入；
6. 防止重复提交同一个行动；

8. 提供减少动画模式；
9. 游戏逻辑与视觉动画完全分离；
10. 即使关闭动画，规则也必须正常运行。

# 二十九、建议项目结构

请按照职责拆分文件，不要把全部逻辑写进一个JS文件。

建议结构：

```text
dark-chess-game/
├─ index.html
├─ README.md
├─ css/
│  └─ style.css
├─ js/
│  ├─ config.js
│  ├─ pieces.js
│  ├─ board.js
│  ├─ rules.js
│  ├─ attacks.js
│  ├─ game-state.js

│  ├─ delayed-threat.js
│  ├─ renderer.js
│  ├─ animations.js
│  ├─ audio.js
│  ├─ input.js
│  └─ main.js
├─ assets/
│  ├─ audio/
│  └─ images/
└─ tests/
   └─ rules-test.html
```

文件职责：

* `config.js`：棋盘尺寸、动画时间和开关；

* `move-transaction.js`：行动快照、提交和回滚；
* `delayed-threat.js`：延迟威胁状态机；
* `renderer.js`：界面渲染；
* `animations.js`：移动、残影、翻面和吃子动画；
* `audio.js`：音效和震动；
* `input.js`：鼠标与触屏操作；
* `main.js`：项目入口；
* `rules-test.html`：规则测试页面。
